서린컴퍼니 V42 관리자 연차·대휴 수정 — 적용 순서

[새 기능]
1. 관리자가 직원의 연차 잔여일수를 원하는 숫자로 직접 수정
2. 관리자가 직원의 대휴 잔여일수를 직접 추가·차감·수정
3. 연차 +1, +0.5, -1 / 대휴 +1.5, +1, -1 빠른 버튼
4. 수정 사유 필수 입력
5. 수정자, 날짜, 종류, 증감일수, 사유 이력 저장
6. 직원 목록에서 연차 잔여와 대휴 잔여를 바로 확인
7. 달력의 연차·반차·반반차 사용과 주말근무·대휴 사용을 자동 계산

[중요 계산]
연차 잔여 = 기본 부여 연차 + 관리자 조정 - 달력 연차 사용
대휴 잔여 = 주말근무 적립(1일당 1.5일) + 관리자 조정 - 대휴 사용

[적용 순서]
1. SQL/V42_관리자연차_대휴수정_이력관리.sql을 Supabase SQL Editor에서 전체 실행
2. GitHub 저장소 Code 화면에서 기존 파일을 V42 파일로 교체
3. 특히 app 폴더, package.json, .github/workflows/build-windows.yml을 교체
4. Actions → Build SeorinCompany Windows Setup → Run workflow
5. 성공 후 Artifacts의 SeorinCompany_Setup_Windows_V42 다운로드
6. ZIP을 풀고 SeorinCompany_Setup_1.1.0.exe 실행
7. 기존 프로그램 위에 설치하면 V42로 업데이트됨

[GitHub 파일 교체 방법]
- 가장 쉬운 방법은 기존 저장소 파일을 삭제하지 않고 같은 이름으로 다시 업로드하는 것입니다.
- GitHub 웹 업로드는 같은 이름 파일을 덮어쓸 수 있습니다.
- .github 폴더 업로드가 안 되면 build-windows.yml을 직접 열고 내용을 V42 파일과 동일하게 수정하세요.

[사용 방법]
직원관리 → 직원의 수정 버튼 → 관리자 연차·대휴 직접 수정
→ 원하는 연차 잔여 / 대휴 잔여 입력 → 수정 사유 입력 → 저장
